BIODATA CRUD PROJECT - SETUP GUIDE
===========================================

PROJECT OVERVIEW:
Your biodata form project has been enhanced with full CRUD (Create, Read, Update, Delete) operations using PHP and MySQL with separate files for each operation.

FILES CREATED:
===========================================

1. config.php
   - Purpose: Database configuration and connection
   - Contains: DB_HOST, DB_USER, DB_PASSWORD, DB_NAME
   - Auto-creates: Database and tables on first run
   - Modify: Database credentials if using different credentials

2. create.php
   - Purpose: Handle INSERT operations (CREATE)
   - Receives: POST request with JSON biodata fields
   - Returns: JSON response with success/error status
   - Security: Uses prepared statements to prevent SQL injection

3. read.php
   - Purpose: Handle SELECT operations (READ)
   - Features: 
     * Single record retrieval by ID
     * All records with pagination
     * Search/filter functionality
   - Returns: JSON formatted biodata records

4. update.php
   - Purpose: Handle UPDATE operations
   - Receives: POST request with ID and fields to update
   - Features: Only updates provided fields, preserves others
   - Returns: Success/error confirmation

5. delete.php
   - Purpose: Handle DELETE operations
   - Receives: POST request with record ID
   - Safety: Requires ID confirmation
   - Returns: Success/error message

6. api.php
   - Purpose: Main API router
   - Handles: Route selection based on action parameter
   - Format: api.php?action=create|read|update|delete

7. index.html (MODIFIED)
   - Changes Made:
     * Added CRUD buttons (View All Records, Add New Record)
     * Added data list table with search and pagination
     * Added View/Edit/Delete action buttons
     * Added modal for viewing full record details
     * Integrated JavaScript for all CRUD operations
     * Removed file upload section (can be re-added)
   - Features:
     * Toggle between form and table view
     * Form population for editing
     * Real-time search
     * Pagination support
     * Success/error message display

8. README.md (CREATED)
   - Comprehensive guide with installation and usage instructions

DATABASE SCHEMA:
===========================================
The application creates a 'biodata' table with:
- id (Primary Key, Auto Increment)
- name, gender, birthdate, birthTime, birthPlace
- religion, country, height, bloodGroup
- education, profession
- fatherName, motherName
- email, phone, address
- created_at, updated_at (Timestamps)

OPERATION FLOW:
===========================================

CREATE:
1. User fills form and clicks "Submit"
2. JavaScript collects form data
3. Sends POST to api.php?action=create with JSON
4. create.php validates and inserts into database
5. Returns success/error message
6. User is shown confirmation and redirected to list

READ:
1. User clicks "View All Records"
2. JavaScript sends GET to api.php?action=read
3. read.php queries database with pagination
4. Returns JSON array of records
5. JavaScript displays in table format
6. User can click "View" for full details

UPDATE:
1. User clicks "Edit" on a record
2. JavaScript fetches record via api.php?action=read&id=X
3. Form populates with record data
4. User modifies fields and clicks "Update"
5. JavaScript sends POST to api.php?action=update
6. update.php updates database
7. Returns confirmation and returns to list

DELETE:
1. User clicks "Delete" on a record
2. Confirmation dialog appears
3. User confirms deletion
4. JavaScript sends POST to api.php?action=delete
5. delete.php removes record from database
6. Returns success message
7. Table refreshes to show updated list

TROUBLESHOOTING:
===========================================
Issue: "Connection failed" error
Solution: Check MySQL is running, update credentials in config.php

Issue: Blank page on first load
Solution: Check PHP error log, ensure PHP is enabled

Issue: Records not saving
Solution: Check browser console, verify database permissions

Issue: Search not working
Solution: Check browser console for errors, verify search terms
