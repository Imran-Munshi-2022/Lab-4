🔌 API ENDPOINTS
════════════════════════════════════════════════════════════════════

Base URL: api.php

Endpoints Available:
  POST   api.php?action=create     → Insert new record
  GET    api.php?action=read       → Get all records (paginated)
  GET    api.php?action=read&id=X  → Get single record
  GET    api.php?action=read&page=1&limit=10 → Pagination control
  POST   api.php?action=update     → Update record
  POST   api.php?action=delete     → Delete record

Response Format (JSON):
{
    "status": "success|error",
    "message": "Description",
    "data": { /* records */ },
    "id": 123,
    "total": 50,
    "page": 1,
    "limit": 10
}

🔄 WORKFLOW DIAGRAM
════════════════════════════════════════════════════════════════════

CREATE Flow:
User fills form → JavaScript collects data → 
POST to api.php → create.php → Database INSERT → 
JSON response → Success message → Redirect to list

READ Flow:
User clicks "View All" → JavaScript sends GET → 
api.php routes to read.php → Database SELECT → 
JSON array returned → Populate table → Display

UPDATE Flow:

api.php routes to update.php → Database UPDATE → 
Success message → Refresh list

🆘 QUICK TROUBLESHOOT
════════════════════════════════════════════════════════════════════

Problem: "Cannot connect to database"
Solution: 
  1. Ensure MySQL is running
  2. Check credentials in config.php
  3. Verify localhost is correct

Problem: "Table already exists"
Solution: 
  1. This is normal on subsequent runs
  2. Application handles this gracefully

Problem: "Records not saving"
Solution:
  1. Check browser console (F12)
  2. Check PHP error logs
  3. Verify file permissions

Problem: "Search not working"
Solution:
  1. Ensure search terms match exactly
  2. Check JavaScript console
  3. Try with different field (name vs email)
